So, what is ES 6?

  • Significant syntax changes
  • No semantic changes
  • Now referred to as ES 2015
  • Finalised in June 2015
  • Move to a 'yearly' release (ES2016, etc)

Browser support

  • Chrome 49 - 91%
  • Firefox 45 - 85%
  • Edge 13 - 83%
  • Mobile? Don't bother..

One javascript

  • Avoid versioning
  • Backwards compatibility
  • Progressive enhancement
  • This is just the beginning.

So what is Babel?

  • Formerly 6to5
  • Javascript-to-javascript compiler
  • Pluggable / composable
  • Build-time, rather than runtime
  • Output can run today

Compiling

  • Source-to-source
  • Transpiling
  • Human readable output
  • Potentially multiple target versions

Pluggable

  • Everything is a plugin
  • Does nothing by default
  • Presets (ES2015, ES2016, react)
  • Choose specific modules (experimental, shims)
  • Write own transpiler step (tricky)

Build time

  • Build tool plugin (cli, grunt, gulp, etc)
  • Environments (browser, node.js, cordova)
  • Frameworks (sails, meteor)
  • Test frameworks

Compelling reasons to use ES 2015

Let + const

						
							// a is hoisted to here
							// b & c aren't

							if (condition) {
								  var a = 1;
								  let b = 1;
								  const c = 1;
								  c = 2; // not updated
							}
  • Block (rather than function) scoped
  • Const is single-assignment

Arrow functions

						
							var sum = values.map(x => x * 2)
							                .reduce((x, y) => x + y);
  • Anonymous function shorthand
  • Lexical this
  • Modern language feature

Template strings

						
							var a = "Centurion",
							    b = "Developer UG",
							    c = "Wednesday";

							var greeting = `Hello ${a} ${b}. Happy ${c}!`;
  • Syntactic sugar
  • Modern language feature

Destructuring

						
							var [a, b] = [1, 2, 3];
							var {c, d} = {
								  c: 42,
								  d: true
							};
  • Pattern matching binding
  • Fails softly (to undefined)

Default, rest, spread

						
							var thingy = function(first = 0, ...args) {
								  /**/
							};

							thingy(...[1,2,3]);
  • No need to use || coalesce hack
  • No need to use arguments array directly
  • Addresses emergent javascript patterns

Classes

						
							class Dog extends Animal {
								  constructor(name) {
									    super(name);
								  }

								  getName() {
									    return this.name;
								  }
							}
  • Doesn't change prototypal inheritence
  • Turned out to be a contentious feature
